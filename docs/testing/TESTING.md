# Документация по тестированию — FileSorted

## Текущее состояние тестов

На данный момент проект **не содержит автоматизированных unit-тестов** в папке `src/test/`.

## Рекомендации по тестированию

1. **Ручное тестирование**:
   - Запустите приложение.
   - Выберите тестовую папку с разными типами файлов.
   - Убедитесь, что файлы перемещены в правильные подпапки.
   - Проверьте поведение при:
     - Пустой папке.
     - Файлах без расширения.
     - Дубликатах имён.
     - Отсутствии прав на запись.

2. **Будущие unit-тесты**:
   - Протестировать метод определения категории файла по расширению.
   - Протестировать создание папок (мокая файловую систему).
   - Использовать `JUnit 5` + `Mockito`.

> Тесты можно разместить в `src/test/java/com/example/sorter/`.
